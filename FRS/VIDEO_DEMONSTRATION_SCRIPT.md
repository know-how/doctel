### SCENE 7: DEMO - INGESTION PROGRESS (4:45-5:45)
**Duration:** 60 seconds

**Visual:**
- Document card now shows "Analyzing..." status
- Progress indicator with percentage
- Step-by-step progress:
  - Extracting text (25%)
  - Chunking content (50%)
  - Generating embeddings (75%)
  - Summarizing and analyzing (90%)
  - Creating prompts (100%)

**Narration:**
```
"Notice the progress indicator on the document card. DocTel shows 
real-time updates as it processes your document:

First, the system extracts text from the document. For PDFs, this 
means reading every page. For Word documents, it extracts all content.

Next, it splits the text into chunks - roughly 1000 tokens each - 
so the AI can understand the content in digestible pieces.

Then, it generates embeddings for each chunk. Embeddings are 
mathematical representations that capture the meaning of text, 
allowing for semantic search later.

Finally, DocTel generates summaries, extracts entities, analyzes 
sentiment, identifies key topics, and creates 3-5 suggested questions.

For a typical 10-page document, this entire process takes 30-60 seconds."
```

**Timing:**
- Show each progress stage for ~15 seconds
- Total duration allows for complete processing

```
The system is doing several things in the background:
1. Converting your question into a mathematical representation (embedding)
2. Searching the vector database for chunks that match your question
3. Ranking results by relevance
4. Selecting the most relevant chunks as context
5. Sending the context and your question to the AI model
6. Receiving and displaying the answer
7. Automatically extracting and showing citations

The entire process - from question to answer with citations - takes 
just a few seconds, but provides intelligent, sourced information."
```

## Recording Instructions

### Desktop Recording Setup
1. **Resolution:** 1920x1080 (Full HD) minimum
2. **Frame Rate:** 60 FPS for smooth animations
3. **Audio:** 48kHz, 16-bit stereo
4. **Tool:** OBS Studio (free), Camtasia, or ScreenFlow

### Recording Checklist
- [ ] Maximize browser window to full screen
- [ ] Hide taskbar/dock during recording
- [ ] Close all notifications
- [ ] Test internet connection (stable)
- [ ] Have sample documents ready:
  - [ ] HSE_Policy_2026.pdf (for demo)
  - [ ] Q1_Report_2026.pdf (for project demo)
- [ ] Use consistent color theme/background
- [ ] Test audio levels before recording
- [ ] Do a practice run first

### Scene Recording Order
1. Record intro graphics separately (can use templates)
2. Record all screen interactions consecutively
3. Record graphics/animations separately
4. Add narration in post-production (easier for editing)
5. Combine all elements in video editor

## Audio Narration Guide

### Voice Talent Specifications
- Professional, clear speaking voice
- Pace: Moderate (125-150 words per minute)
- Tone: Friendly but professional
- Accent: Clear and understandable
- No heavy background noise

### Recording Narration
1. Read full script aloud to time it (should be ~14-15 minutes)
2. Record in quiet room with good microphone
3. Do multiple takes for each scene
4. Edit best takes together
5. Add background music at 30-40% volume

### Background Music Selection
- Tech/corporate theme (for intro)
- Uplifting background during features (20-30% volume)
- Calm during demonstrations
- Energetic conclusion
- Royalty-free sources: Bensound, AudioJungle, Epidemic Sound

## Post-Production Guidelines

### Video Editing
1. Arrange scenes in order
2. Add transitions between scenes (0.5-1 second)
3. Add text overlays for key points
4. Insert graphics/animations
5. Sync narration with on-screen action
6. Add captions for accessibility

### Color Grading
- Match DocTel brand colors (blue #5B88FF, cyan #1FE7FF)
- Maintain professional, modern aesthetic
- Ensure text readability on all backgrounds

### Subtitles/Captions
- Essential for accessibility
- Time to match narration
- Include speaker identification
- Use large, readable font

### Export Settings
- Format: MP4 or WebM
- Codec: H.264 or VP9
- Bitrate: 5-8 Mbps (balance quality/file size)
- Resolution: 1920x1080 (or scale to 4K if needed)
- Frame rate: 60 FPS

### File Naming
`DocTel_FRS_Demo_v1.0_1080p.mp4`
